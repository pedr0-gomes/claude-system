Fundamentos de git/github fechados just-in-time (2026-06-13), recontados
com as próprias palavras e grelhados contra o histórico real de um repo.
**Domina:** versionamento (commit = foto recuperável, não sobrescrita),
staging (`git add`), local vs remoto, git ≠ github (github como remote +
encontro entre pessoas + `private` como tranca), caminho de volta
(`clone` vs `pull`).
**Ainda não:** `status`/`diff`/`log`/desfazer (dia a dia), branch, merge,
conflitos, pull request.

Conteúdo canônico (explicações + mapa "pra depois") na nota-sombra do
projeto: `~/dev/projects/organograma-enfermagem-hmsvp/notas/git-github-versionamento.md`.
Calibra a profundidade de explicação de git em qualquer projeto.
